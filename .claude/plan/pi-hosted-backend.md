## Locked decisions (from clarifying Q&A)

| Q | Choice | Why it matters |
|---|---|---|
| Process shape on Pi | **One FastAPI process; hardware loop = asyncio background task via `lifespan`** | One systemd unit, one log stream, one ngrok tunnel. Internal calls bypass HTTP. |
| Frontend data path | **Supabase direct for reads + ngrok→Pi for control actions** | Dashboard survives Pi offline. Only "dispense now" / status pings hit the Pi. |
| ngrok plan | **Free tier, rotating URL** | Cheapest. Operator updates `NEXT_PUBLIC_DEVICE_URL` after each Pi reboot. |
| Delete `edge_pi/` | **Yes, full merge into `backend/`** | Architecture collapses 3-tier → 2-tier (Pi-as-backend + Frontend). |

## What changes (architectural diff)

### Before
```
backend/      stateless FastAPI, runs anywhere, talks to Supabase
edge_pi/      polling client, runs on Pi, hits backend over HTTP
frontend/     Next.js, talks Supabase + occasionally backend
```

### After
```
backend/      FastAPI on Pi: HTTP API + asyncio hardware loop in ONE process
              (no more self-HTTP for adherence; calls service funcs directly)
              Exposed externally via ngrok tunnel.
frontend/     Next.js, talks Supabase for reads + ngrok→Pi for control actions
edge_pi/      DELETED
```

## Patterns to mirror

| Pattern | Source | Notes |
|---|---|---|
| FastAPI lifespan with background task | `https://fastapi.tiangolo.com/advanced/events/#lifespan` | `@asynccontextmanager` + `app.state.<task>` |
| Reference flat layout | `medispecs-backend-specs/main.py` + `routers/` + `services/` + `oled_device.py` | Hardware drivers at top level. main.py at root, not under app/. |
| Existing PharmGuard cycle | `edge_pi/main.py:run()` (the `while True` body) | Whole body becomes an async function in `scheduler/cycle_runner.py` |
| Stub-mode invariant | `edge_pi/hardware/*.py:STUB_ALLOWED` + `edge_pi/main.py:301-314` | HI-012: stub mode never logs `pill_taken=true`. Carry forward verbatim. |
| Offline queue | `edge_pi/storage/queue.py` + `edge_pi/main.py:offline_queue` | Stays in-process. Replays at top of each cycle. |

## Step-by-step tasks

### Task 1: Branch + safety net
- ACTION: Create feature branch `feat/pi-hosted-backend` from `main`. Tag current `main` as `pre-merge-snapshot`.
- VALIDATE: `git tag -l pre-merge-snapshot` shows the tag.

### Task 2: Move trees (hardware / vision / storage / scripts / tests / models)
- ACTION: `git mv edge_pi/hardware backend/hardware`; same for `vision`, `storage`, `scripts`, `tests`, `models`.
- GOTCHA: These modules use absolute imports like `from hardware.magazine import Magazine`. Those keep working as long as `backend/` is the working directory and is on `sys.path`. Confirm with `python -c "from hardware.magazine import Magazine"` from `backend/`.
- VALIDATE: `find backend/hardware -name '*.py' | xargs python -m py_compile` passes from `backend/`.

### Task 3: Flatten `backend/app/` → `backend/`
- ACTION: `git mv backend/app/api backend/api`; same for `services`, `db`, `core`. Move `backend/app/main.py` → `backend/main.py`. Remove `backend/app/`.
- GOTCHA: All `from app.x import y` references in the moved files must become `from x import y` (drop the `app.` prefix). Run a single sed pass across moved files. Verify no `app.` import remains: `grep -RnE "from app\.|import app\." backend/`.
- VALIDATE: `cd backend && python -c "import main"` succeeds (with stub env).

### Task 4: Merge configs
- ACTION: New `backend/config.py` exposes ONE settings object. Keys merge:
  - From old backend `Settings`: `SUPABASE_URL`, `SUPABASE_KEY`, `GEMINI_API_KEY`, `EXPIRY_WARN_DAYS`, `LOW_STOCK_THRESHOLD`, `OVER_TEMP_CELSIUS`, `FACE_MATCH_TOLERANCE`.
  - From old edge_pi `_Settings`: `POLL_INTERVAL_S`, `DISPENSER_ID`, `BENCH_MODE`, `BENCH_LOG_PATH`, `OFFLINE_QUEUE_PATH`, `OFFLINE_MAX_AGE_SECONDS`, `OFFLINE_REPLAY_INTERVAL_S`.
  - DROP: `BACKEND_URL`, `DEVICE_TOKEN` (no longer needed — Pi IS the backend).
  - ADD: `DEVICE_API_KEY` (shared secret for frontend → Pi auth via ngrok), `BACKEND_HEADLESS` (1 = skip hardware lifespan, for dev-mac runs).
- MIRROR: keep `pydantic-settings` pattern from old `backend/app/core/config.py`.
- VALIDATE: `cd backend && python -c "from config import settings; print(settings)"` after seeding `.env`.

### Task 5: Refactor cycle into `scheduler/cycle_runner.py`
- ACTION: Move the body of `edge_pi/main.py:run()` into `async def run_cycle(state: CycleState) -> None`. Keep the per-step `time.perf_counter()` instrumentation. Replace blocking `time.sleep(...)` with `await asyncio.sleep(...)`. GPIO/cv2 calls stay synchronous — wrap them in `await asyncio.to_thread(...)` so they don't block FastAPI's event loop.
- KEY transformation:
  ```python
  # before (edge_pi/main.py)
  resp = session.post(f"{BACKEND_URL}/api/auth/verify-face", files={...})
  # after (backend/scheduler/cycle_runner.py)
  from services.face_recognition import verify_face_bytes
  patient = await asyncio.to_thread(verify_face_bytes, crop_bytes)
  ```
- GOTCHA: HI-012 stub-mode guard at `edge_pi/main.py:301-314` is load-bearing. Copy verbatim. The merged process MUST refuse to log `pill_taken=true` when any hardware is stubbed.
- VALIDATE: import-only smoke `python -c "from scheduler.cycle_runner import run_cycle"`.

### Task 6: HardwareLoop supervisor (`scheduler/background.py`)
- ACTION: Class with `start()` / `stop()` / `trigger_dispense_now()`. `start()` spawns `asyncio.create_task(self._supervised_loop())` which runs `run_cycle` forever, restarting on exception with exponential backoff (cap 60 s). `stop()` cancels the task and awaits cleanup of GPIO/cameras.
- IMPORTS: `asyncio`, `logging`, `scheduler.cycle_runner.run_cycle`.
- GOTCHA: A bare `lifespan`-level crash kills FastAPI. Catch broad exceptions inside `_supervised_loop`; only fatal init errors (e.g. GPIO refuses to init AND stub disallowed) propagate to lifespan and abort startup.
- VALIDATE: unit test in `tests/test_background.py` that asserts the loop restarts after a synthetic `RuntimeError`.

### Task 7: Wire `lifespan` into FastAPI `main.py`
- ACTION:
  ```python
  from contextlib import asynccontextmanager
  from scheduler.background import HardwareLoop
  from config import settings

  @asynccontextmanager
  async def lifespan(app: FastAPI):
      if settings.BACKEND_HEADLESS:
          log.info("BACKEND_HEADLESS=1 — skipping hardware loop init")
          yield
          return
      loop = HardwareLoop()
      await loop.start()
      app.state.hardware_loop = loop
      try:
          yield
      finally:
          await loop.stop()

  app = FastAPI(lifespan=lifespan, title="PharmGuard", version="0.2.0")
  ```
- ROUTERS: include the existing four (`auth`, `inventory`, `logs`, `alerts`) + the new `device` router.
- VALIDATE: `BACKEND_HEADLESS=1 uvicorn main:app --port 8000` starts cleanly on dev-mac (no GPIO).

### Task 8: New `api/device.py` router
- ACTION: Three endpoints, all behind `Depends(require_device_api_key)`:
  - `POST /api/device/dispense_now` — calls `app.state.hardware_loop.trigger_dispense_now()`. Returns 202 + cycle id.
  - `GET /api/device/status` — returns process state, last cycle metrics, current slot, stub flags.
  - `POST /api/device/reset` — admin only; cancels in-flight cycle, returns drawer to locked, magazine homes.
- AUTH: `require_device_api_key` reads `X-Device-API-Key` header, compares to `settings.DEVICE_API_KEY`. 401 on mismatch.
- VALIDATE: `curl -H "X-Device-API-Key: $KEY" http://localhost:8000/api/device/status` returns 200 with stub state.

### Task 9: Merge `requirements.txt`
- ACTION: New `backend/requirements.txt` = union of old backend's + edge_pi's. Pin ranges identical. Remove duplicates.
- GOTCHA: `face_recognition` (backend) + `mediapipe` (edge_pi) + `picamera2` (edge_pi) + `rpi-lgpio` (edge_pi) + `adafruit-circuitpython-dht` (edge_pi) all coexist. mediapipe needs cp ≤ 3.12. picamera2 + rpi-lgpio are Pi-only — Mac venv will fail to install picamera2; gate it with `picamera2; platform_machine == 'aarch64'` markers.
- VALIDATE: `pip install --dry-run -r backend/requirements.txt` from a fresh aarch64 venv.

### Task 10: systemd + ngrok services
- ACTION: Update `backend/scripts/pharmguard.service` ExecStart to `uvicorn main:app --host 127.0.0.1 --port 8000 --workers 1` (workers=1 mandatory — multi-worker forks corrupt GPIO state). WorkingDirectory becomes `__INSTALL_DIR__` = `backend/`. Add new `backend/scripts/ngrok.service` running `ngrok http --log=stdout 8000` after `pharmguard.service` is up. Add `backend/scripts/install.sh` updates: install both units, prompt for ngrok auth token.
- GOTCHA: Free ngrok URL changes every restart. Add a post-start script that writes the current public URL to `/var/lib/pharmguard/ngrok_url.txt` so the operator (or a watcher) can read it.
- VALIDATE: `systemctl status pharmguard ngrok` both `active (running)` after install on Pi.

### Task 11: Frontend `lib/device.ts` + Dispense-Now button

## Risks & mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Multi-worker uvicorn corrupts GPIO state | M | H (silent dispense failures) | Pin `--workers 1` in pharmguard.service, add startup check. |
| Hardware loop crash propagates to FastAPI lifespan, kills HTTP API | M | H (dashboard goes dark on Pi-side crash) | `_supervised_loop` catches broad exceptions, exponential restart. Only `init failure + stub disallowed` aborts lifespan. |
| Free ngrok URL changes break frontend | H | M (manual env update needed) | Document in README. Long-term: paid static domain. Mid-term: small "URL bulletin" page hosted on Vercel/static that the ngrok daemon writes to on start. |
| `NEXT_PUBLIC_DEVICE_API_KEY` is a soft key (visible in browser) | H | M (anyone scraping the bundle can dispense) | Document explicitly. Recommend Supabase Edge Function proxy as follow-up. |
| Dev-mac venv breaks on picamera2 / rpi-lgpio | H | L (annoying) | `; platform_machine == 'aarch64'` pip markers + `BACKEND_HEADLESS=1` skip flag. |
| HI-012 invariant gets lost during refactor | M | H (falsified adherence) | Carry stub-mode guard verbatim from `edge_pi/main.py:301-314`. Add explicit unit test asserting stub mode → no `pill_taken=true` log. |
| In-flight refactor breaks both backends at once (no fallback) | M | M (Pi can't dispense, dashboard reads break) | Keep `pre-merge-snapshot` git tag. Branch is `feat/pi-hosted-backend` — main stays working until merge. |
| Self-HTTP latency saved (~20 ms) is offset by asyncio.to_thread overhead | L | L | Benchmark before/after with `bench_e2e.py` after refactor. |
| Workers=1 means single connection saturates the API | L | L (low traffic — caregiver dashboard only) | Acceptable for v1; revisit with metrics. |

## Confidence

**5 / 10** for single-pass implementation.

Reasons for caution:
- ~50 file moves with import-rewriting — every misnamed import is a hidden runtime error.
- New asyncio supervisor is non-trivial; HardwareLoop crash semantics need real Pi to validate.
- ngrok rotation is a real ergonomic problem the user has accepted but will hit immediately.
- HI-012 invariant is the safety boundary; if any of the moved code drops the stub-mode check the system silently falsifies adherence.

Mitigation: implement on a feature branch (`feat/pi-hosted-backend`), keep `pre-merge-snapshot` git tag for instant rollback, validate each task's checkpoint before moving on. Per-task validation gates (above) are NON-OPTIONAL.
